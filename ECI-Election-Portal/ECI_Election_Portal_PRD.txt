1. PRODUCT OVERVIEW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Product Name    : ECI Voter Portal (Bharat Nirwachan Portal)
Tagline         : Your Vote. Your Democracy. Powered by Intelligence.
Owner           : Election Commission of India (ECI)
Primary Users   : Indian citizens (voters), journalists, researchers, candidates
Design System   : IBM Carbon Design System (modified — ECI tricolor branding)
Core Purpose    : Provide a single authoritative platform for voter registration,
                  election education, AI-assisted civic Q&A, real-time election
                  data, candidate discovery, grievance filing, and constituency
                  exploration for India's 970M+ eligible voters.

  Primary Font     : IBM Plex Sans (300, 400, 600)
  Monospace Font   : IBM Plex Mono (400) — stats, codes, tables
  Display Hero     : 60px / weight 300 / letter-spacing 0
  Section Titles   : 32px / weight 400
  Card Titles      : 20px / weight 600
  Body             : 16px / weight 400 / line-height 1.5
  Labels/Captions  : 12px / weight 400 / letter-spacing 0.32px

3.3 LAYOUT RULES
  Max Content Width   : 1312px centered with 32px gutters
  Grid System         : 16-column CSS Grid (Carbon 2x Grid)
  Border Radius       : 0px everywhere (Carbon identity — no rounded corners)
  Shadows             : None — depth via background layering only
  Button Height       : 48px (primary), 40px (secondary)
  Input Height        : 40px, bottom-border only (no box border)

3.4 COMPONENT PATTERNS
  Navigation    : 48px dark (#161616) sticky header, tricolor 4px top bar
  Tiles         : Flat cards on layer-01 bg, 1px gap-grid layout
  Tables        : Carbon data table — 2px bottom border on header
  Forms         : Bottom-border inputs on dark (#161616) background sections
  Tabs          : Bottom-border active indicator in IBM Blue
  Badges/Pills  : 24px border-radius colored status pills
  Notifications : Left-border inline notification (4px colored left border)

3.5 RESPONSIVE BREAKPOINTS
  Desktop    : 1312px+  (full 4-column quick actions, 3-column tiles)
  Tablet     : 672–1056px (2-column tiles, 3-column timeline)
  Mobile     : < 672px  (single column, hidden nav links, no hero card)

FEATURE 2: AI-POWERED Q&A CHATBOT
──────────────────────────────────────────────────────────────────────────────
Description  : Floating AI assistant powered by Gemini 1.5 Flash with RAG
               over election law, voter rights, and ECI guidelines documents.
               Answers questions in English and Hindi.

UI Elements  :
  - Floating chat bubble (bottom-right, IBM Blue, 56px circle)
  - Slide-up chat panel (400px wide, Carbon dark theme)
  - Message bubbles: user (saffron bg), AI (layer-02 bg)
  - Suggested prompt chips: "How do I register?", "When is polling day?", etc.
  - Language toggle: English / हिन्दी
  - Source citations shown as small pills below AI responses
  - Typing indicator (3 animated dots)
  - Clear conversation button
  - Rate limit: 20 messages/hour for unauthenticated, unlimited for logged-in

Tech Details :
  - Python FastAPI endpoint: POST /ai/chat
  - Gemini 1.5 Flash for response generation
  - RAG pipeline: election law PDFs → chunked → Gemini embeddings → pgvector
  - System prompt includes: ECI guidelines, Model Code of Conduct, voter rights
  - Streaming response via Server-Sent Events (SSE)
  - Conversation history stored in Redis (TTL: 1 hour)

Backend API  :
  POST /api/ai/chat            { message, history[], language }
  GET  /api/ai/suggestions     Returns contextual prompt suggestions

DB Tables    : chat_sessions, document_embeddings (pgvector)

FEATURE 4: COMPLAINT / GRIEVANCE PORTAL
──────────────────────────────────────────────────────────────────────────────
Description  : Digital version of cVIGIL — file, track, and escalate
               complaints about electoral malpractice, voter intimidation,
               bribery, and MCC violations.

UI Elements  :
  - Grievance form (Google OAuth required):
      * Complaint category (MCC Violation / Voter Intimidation / Booth Capture
        / Bribery / Fake News / Other)
      * Description textarea (min 50 chars)
      * Location picker (state → district → constituency)
      * Evidence upload (photo/video, max 50MB, to Supabase Storage)
      * Anonymous filing option (checkbox)
  - Submission confirmation with 12-char ticket ID (IBM Plex Mono)
  - Track Status page:
      * Enter ticket ID → see current status
      * Status timeline: Filed → Under Review → Investigated → Resolved
      * ECI officer comments (if any)
  - Admin view: grievance queue with priority scoring (AI-assigned)

AI Integration :
  - Gemini classifies complaint severity (High/Medium/Low) on submission
  - Auto-generates summary for ECI officer review
  - Duplicate detection: embeddings similarity check against last 30 days

Backend API  :
  POST /api/grievance/submit
  GET  /api/grievance/track/:ticketId
  GET  /api/grievance/list           (admin only)
  PATCH /api/grievance/:id/status    (admin only)

DB Tables    : grievances, grievance_evidence, grievance_timeline

FEATURE 5: VOTER TURNOUT PREDICTIONS
──────────────────────────────────────────────────────────────────────────────
Description  : Real-time dashboard showing live turnout numbers during
               polling hours + Gemini-powered predictions by state/constituency
               using historical data and demographic signals.

UI Elements  :
  - Live Turnout Tracker section:
      * Phase progress bar (saffron fill on dark bg)
      * India-level aggregate turnout % (giant IBM Plex Mono number)
      * State-wise turnout table (sortable Carbon data table)
      * Last updated timestamp (auto-refreshes every 60s via Socket.IO)
  - Prediction Dashboard:
      * Recharts area chart: actual vs predicted turnout per hour
      * State-wise prediction cards (color-coded: green >65%, yellow 50-65%, red <50%)
      * Demographic breakdown: urban vs rural, male vs female
      * "Factors affecting turnout" — Gemini-generated insight card
  - Historical Comparison:
      * Bar chart: 2014 / 2019 / 2024 / 2026 turnout by phase

Tech Details :
  - Python FastAPI: ML prediction model (sklearn/xgboost trained on historical data)
  - Gemini 1.5 Pro: narrative insights from prediction data
  - Supabase Realtime: live turnout numbers pushed by ECI data pipeline
  - Socket.IO: push updates to connected clients (no polling)
  - BullMQ job: scrapes ECI turnout API every 5 minutes during polling hours

Backend API  :
  GET  /api/turnout/live             Returns live turnout by state
  GET  /api/turnout/predictions/:state
  GET  /api/turnout/historical
  WS   /socket/turnout-updates       Socket.IO channel

DB Tables    : turnout_snapshots, turnout_predictions, historical_turnout

6. DATABASE SCHEMA (SUPABASE / POSTGRESQL)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

TABLE: voters
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid()
  epic_number     VARCHAR(10) UNIQUE NOT NULL
  full_name       VARCHAR(200) NOT NULL
  dob             DATE NOT NULL
  gender          VARCHAR(20)
  address         TEXT
  state           VARCHAR(100)
  constituency_id UUID REFERENCES constituencies(id)
  booth_id        UUID REFERENCES polling_booths(id)
  is_active       BOOLEAN DEFAULT true
  created_at      TIMESTAMPTZ DEFAULT now()

TABLE: candidates
  id              UUID PRIMARY KEY
  name            VARCHAR(200)
  party_id        UUID REFERENCES parties(id)
  constituency_id UUID REFERENCES constituencies(id)
  phase           INTEGER
  photo_url       TEXT  -- Supabase Storage URL
  assets_declared BIGINT  -- in INR
  criminal_cases  INTEGER DEFAULT 0
  education       VARCHAR(200)
  affidavit_url   TEXT  -- Supabase Storage PDF URL
  ai_summary      TEXT  -- Gemini generated
  created_at      TIMESTAMPTZ DEFAULT now()

TABLE: candidate_social
  id              UUID PRIMARY KEY
  candidate_id    UUID REFERENCES candidates(id)
  platform        VARCHAR(50)  -- twitter, facebook, instagram, website
  url             TEXT
  handle          VARCHAR(200)

TABLE: parties
  id              UUID PRIMARY KEY
  name            VARCHAR(200)
  abbreviation    VARCHAR(20)
  logo_url        TEXT
  color_hex       VARCHAR(7)  -- for UI accent strip

TABLE: constituencies
  id              UUID PRIMARY KEY
  name            VARCHAR(200)
  state           VARCHAR(100)
  type            VARCHAR(20)  -- lok_sabha | assembly
  phase           INTEGER
  geojson         JSONB  -- constituency boundary

TABLE: polling_booths
  id              UUID PRIMARY KEY
  constituency_id UUID REFERENCES constituencies(id)
  name            VARCHAR(300)
  address         TEXT
  lat             DECIMAL(10,7)
  lng             DECIMAL(10,7)
  is_accessible   BOOLEAN DEFAULT true
  location        GEOGRAPHY(POINT, 4326)  -- PostGIS

TABLE: grievances
  id              UUID PRIMARY KEY
  ticket_id       VARCHAR(12) UNIQUE  -- public-facing ID
  user_id         UUID REFERENCES auth.users(id)
  category        VARCHAR(100)
  description     TEXT
  constituency_id UUID
  is_anonymous    BOOLEAN DEFAULT false
  ai_severity     VARCHAR(20)  -- High / Medium / Low (Gemini assigned)
  ai_summary      TEXT
  status          VARCHAR(50) DEFAULT 'filed'
  created_at      TIMESTAMPTZ DEFAULT now()

TABLE: grievance_evidence
  id              UUID PRIMARY KEY
  grievance_id    UUID REFERENCES grievances(id)
  file_url        TEXT  -- Supabase Storage
  file_type       VARCHAR(50)
  uploaded_at     TIMESTAMPTZ DEFAULT now()

TABLE: grievance_timeline
  id              UUID PRIMARY KEY
  grievance_id    UUID REFERENCES grievances(id)
  status          VARCHAR(50)
  note            TEXT
  officer_id      UUID
  timestamp       TIMESTAMPTZ DEFAULT now()

TABLE: turnout_snapshots
  id              UUID PRIMARY KEY
  constituency_id UUID
  state           VARCHAR(100)
  phase           INTEGER
  recorded_at     TIMESTAMPTZ
  turnout_pct     DECIMAL(5,2)
  total_votes     INTEGER

TABLE: news_articles
  id              UUID PRIMARY KEY
  source_id       UUID REFERENCES news_sources(id)
  headline        TEXT NOT NULL
  summary         TEXT  -- Gemini generated
  url             TEXT UNIQUE
  category        VARCHAR(100)
  fact_check      VARCHAR(50)  -- verified | unverified | disputed
  published_at    TIMESTAMPTZ
  state_tag       VARCHAR(100)
  phase_tag       INTEGER
  embedding       vector(768)  -- pgvector for semantic search

  id              UUID PRIMARY KEY

  embedding       vector(768)

8. AUTHENTICATION FLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. User clicks "Sign in with Google" → Google OAuth 2.0 flow
2. Google callback → Supabase Auth handles session
3. Supabase issues JWT (15 min TTL) + refresh token (30 days)
4. Frontend stores JWT in memory, refresh token in httpOnly cookie
5. Every API request: Authorization: Bearer <jwt>
6. On 401: auto-refresh using cookie-based refresh token
7. Roles assigned in Supabase user_metadata: voter | admin | journalist

Features requiring auth: Grievance filing, Voter profile, Admin dashboard
